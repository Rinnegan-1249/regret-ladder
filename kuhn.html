Kuhn Poker

The smallest real poker game — private cards, betting, and bluffing in equilibrium. Play it, watch four CFR-family algorithms train, read the theory.

Rules

Kuhn Poker (Kuhn 1950) uses a three-card deck: Jack < Queen < King. Both players ante 1 chip; each is dealt one private card. A single betting round follows with bet size 1:

Despite its size (58 game-tree nodes, 12 information sets), Kuhn Poker has everything that makes poker interesting: hidden information, and a Nash equilibrium that requires bluffing and mixed strategies.

The Nash equilibrium family

Kuhn Poker has a one-parameter family of Nash equilibria, indexed by \(\alpha \in [0, \tfrac{1}{3}]\): Player 1 opens with a bet holding the Jack (a pure bluff) with probability \(\alpha\), holding the King with probability \(3\alpha\), and never with the Queen. The game value is \(-\tfrac{1}{18}\) chips/hand for Player 1. Our CFR+ run recovers a member of this family exactly: \(\alpha = 0.2226\), \(P(\text{bet}\,|\,K) = 0.6678 = 3\alpha\) (see Results).

Things to look for: with the Jack, Player 1's opening bet probability settles at some \(\alpha \in [0, 1/3]\) (a deliberate bluff!), and with the King at exactly \(3\alpha\); the Queen never opens. CFR+'s exploitability dives an order of magnitude below vanilla CFR's at the same iteration count; the MCCFR curves are noisy (each iteration touches only a sampled part of the tree).

Vanilla CFR (Zinkevich et al. 2007 [1])

CFR traverses the full game tree each iteration, computing at every information set \(I\) the counterfactual values \(v_i(\sigma^t, I)\) and \(v_i(\sigma^t_{(I \to a)}, I)\) (see the Regret section), accumulating regrets, and setting the next strategy by regret matching. By Theorem 2 (regret decomposition) the overall regret is bounded by the per-information-set sums, and Theorem 1 turns vanishing regret into an approximate Nash equilibrium of the average strategy, with bound \(R^T_i \le \Delta_{u,i}\, M_i \sqrt{|A_i|}/\sqrt{T}\). Our implementation uses simultaneous updates with frozen regret tables and is validated against OpenSpiel's CFRSolver.

CFR+ (Tammelin 2014 [3])

Three changes, an order of magnitude faster in practice:

  1. Regret-matching+ — cumulative regrets are floored at zero each update:
    $$R^{+,T}(I,a) = \max\!\big\{R^{+,T-1}(I,a) + v_i(\sigma^T_{I\to a}, I) - v_i(\sigma^T, I),\; 0\big\},$$
    so an action that has been bad for a long time can come back immediately when it turns good (in CFR it must first dig out of a deep negative-regret hole).
  2. Alternating updates — each iteration updates one player at a time.
  3. Weighted averaging — iteration \(t\)'s strategy enters the average with weight \(w_t = \max(t - d,\, 0)\) (linear weighting; \(d\) is a delay, 0 here).

Empirically CFR+'s current strategy also (nearly) converges — vanilla CFR's never does — which we reproduce in the live-training tab and in Results.

Monte Carlo CFR (Lanctot et al. 2009 [2])

MCCFR avoids full tree traversals: partition the terminal histories \(Z\) into blocks \(\mathcal{Q} = \{Q_1, \dots, Q_r\}\), sample a block with probability \(q_j\), and use the sampled counterfactual value

$$\tilde{v}_i(\sigma, I\,|\,j) \;=\; \sum_{z \in Q_j \cap Z_I} \frac{1}{q(z)}\, u_i(z)\, \pi^{\sigma}_{-i}(z[I])\, \pi^{\sigma}(z[I], z),$$

which is unbiased: \(\mathbb{E}_j[\tilde{v}_i(\sigma, I|j)] = v_i(\sigma, I)\) (their Lemma 1) — so the regret updates match CFR in expectation while touching only a fraction of the tree.

  • Outcome sampling: each block is a single terminal history, sampled with an \(\varepsilon\)-greedy profile at the update player (\(\varepsilon = 0.6\), keeping \(q(z) \ge \delta > 0\)) and on-policy elsewhere. The \(1/\delta\) factor appears in its regret bound (Theorem 5). One iteration touches ~10 nodes on Kuhn.
  • External sampling: sample only the choices external to the update player (chance, opponent), enumerate the player's own actions. Since \(q(z) = \pi^\sigma_{-i}(z)\), the counterfactual weights cancel exactly, leaving \(\tilde{r}(I,a) = u(a) - u\). Needs only a constant factor more iterations than CFR at \(O(\sqrt{|H|})\) cost per iteration (Theorem 4) — an asymptotic win in large games.
Honest caveat (see Results): on Kuhn's 58-node tree a full traversal is nearly free, so the sampling variance buys nothing — MCCFR ties or loses to vanilla CFR per node touched here. The paper's dramatic wins come from games with \(10^7\)–\(10^8\) histories. That is the motivation for scaling to Leduc Poker next.
